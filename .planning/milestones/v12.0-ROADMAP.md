# Milestone v12.0: Language Ergonomics & Open Source Readiness

**Status:** ✅ SHIPPED 2026-02-27
**Phases:** 116-125
**Total Plans:** 24

## Overview

Made Mesh more ergonomic for real programs and ready for open source with slot pipe operator, string interpolation, heredocs, regex, env var stdlib, Mesher dogfooding, a Mesh agent skill, repository reorganization, performance benchmarks (29,108 req/s — within 4% of Go), developer seed migration, and updated public documentation.

## Phases

### Phase 116: Slot Pipe Operator

**Goal**: Users can pipe expressions to any argument position using `|N>` syntax with full type checking
**Depends on**: Phase 115
**Requirements**: PIPE-01, PIPE-02, PIPE-03, PIPE-04
**Plans**: 2 plans

Plans:
- [x] 116-01-PLAN.md — Lexer + Parser + AST: SlotPipe token, SLOT_PIPE_EXPR node, SlotPipeExpr with slot() accessor
- [x] 116-02-PLAN.md — Type checker + MIR lowering: infer_slot_pipe, error variants, E2E tests

**Details:** SlotPipe(u32) token emitted for `|N>` (N >= 2). SLOT_PIPE_EXPR CST node with SlotPipeExpr AST exposing lhs/slot/rhs accessors. infer_slot_pipe inserts lhs at position N in callee arg list. SlotPipeOutOfRange error emitted when slot > known arity. Chains with regular pipe at equal precedence (3, 4). Completed 2026-02-25.

### Phase 117: String Interpolation & Heredocs

**Goal**: Users can embed expressions directly in strings with `#{expr}` and write multiline strings with `"""..."""`
**Depends on**: Phase 116
**Requirements**: STRG-01, STRG-02, STRG-03
**Plans**: 2 plans

Plans:
- [x] 117-01-PLAN.md — Lexer: #{...} interpolation delimiter, backward compat with ${...}, E2E tests for STRG-01
- [x] 117-02-PLAN.md — MIR lowerer: trimIndent for triple-quoted heredocs, E2E tests for STRG-02 and STRG-03

**Details:** Both `${ }` and `#{ }` syntaxes emit identical InterpolationStart tokens — parser requires zero changes. apply_heredoc_content() processes each STRING_CONTENT segment for MIR lowering. trimIndent() on heredocs strips common leading whitespace. Completed 2026-02-26.

## Milestone Summary

**Key Decisions:**

- [Phase 116]: |0> and |1> emit TokenKind::Error at lex time (hard error by design)
- [Phase 116]: SlotPipe uses same Pratt binding power (3, 4) as Pipe — chain with equal precedence
- [Phase 116]: Slot pipe uses insertion semantics (x |2> f(a,b,c) = f(a,x,b,c)); SlotPipeOutOfRange emitted when slot > arity
- [Phase 117]: Both ${ and #{ emit identical InterpolationStart tokens — parser requires zero changes
- [Phase 118]: Old bare env_get (Option-returning) removed; env_get now 2-arg with default
- [Phase 118]: env_get_int silently returns default on any parse failure (no stderr warning)
- [Phase 119]: Regex.is_match not Regex.match ('match' is a Mesh keyword)
- [Phase 119]: Ty::Con(Regex) maps to MirType::Ptr — opaque heap pointer
- [Phase 119]: Regex added to STDLIB_MODULE_NAMES
- [Phase 120]: HTTP test assertions fixed: unescape_string() correctly processes backslash escapes
- [Phase 121]: Skill lives at skill/mesh/ with SKILL.md format
- [Phase 122]: crates/ flattened directly to compiler/ (no intermediate directory)
- [Phase 123]: Isolated benchmark: Mesh 29,108 req/s vs Go 30,306 vs Rust 46,244 vs Elixir 12,441
- [Phase 124]: projects.slug ON CONFLICT requires WHERE slug IS NOT NULL for partial index match
